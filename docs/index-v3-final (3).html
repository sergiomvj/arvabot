📖 Manual do Sistema — OpenClaw Dashboard v2

Este dashboard é o centro de controle dos agentes ARVA. Aqui você gerencia identidades, memórias, segurança e evolução de cada agente. Use este manual como referência rápida.

🃏 Anatomia do Card de Agente
Header do Card
Exibe o avatar emoji, nome, papel/role e status (Online/Offline). O ring à direita é o Humanization Score (0–100) — mede o quanto o agente conhece a pessoa que atende. Score verde (≥60) = agente bem contextualizado. Âmbar (30–59) = em evolução. Vermelho (<30) = onboarding pendente.
Métricas (Heartbeat · Uptime · Tarefas)
Faixa com dados operacionais em tempo real. Heartbeat = último sinal de vida do agente. Uptime = disponibilidade histórica. Tarefas = total atribuído ao agente.
Arquivos de Memória (pills clicáveis)
Cada pill representa um arquivo markdown no workspace do agente. Clique para editar diretamente.
✨ SOUL
Personalidade, valores, protocolos SAPP e SMT. O "cérebro" do agente.
🪪 IDENTITY
Cartão de identidade — competências, línguas, background.
🧠 MEMORY
Memória de curto prazo. Atualizada pelo agente após cada sessão.
🧠 MEMORY_MID
Médio prazo — projetos, padrões, stakeholders. TTL: 30–180 dias.
🧠 MEMORY_LONG
Longo prazo — fatos consolidados permanentes sobre o owner.
📈 EVOLUTION
Jornada de evolução IRO — o que mudou desde o THE CALL.
💜 HUMANIZED
Perfil humano da pessoa atendida. Alimentado via onboarding.
🔒 SOUL +SAPP
Protocolo de segurança embutido no SOUL — níveis N1/N2/N3.
👥 AGENTS
Mapa do ecossistema — outros agentes e como colaborar.
🔧 TOOLS
Ferramentas disponíveis + perímetro SAPP + tools SMT.
✓ TASKS
Protocolos de execução e Execution States em andamento (SMT).
💓 HEARTBEAT
Status operacional interno. Não entra no system prompt do agente.
SMT · Uso de Contexto
Barras mostrando o quanto de cada camada de memória está ocupado. Short-term é sempre carregado. Mid-term carrega quando relevante. Long-term raramente carrega — só para referências históricas. Se Short-term ultrapassar 80%, o agente entra em modo OVERFLOW_WARNING e comprime automaticamente.
IRO · Evolução
O ring âmbar mostra o score de padrões confirmados pelo sistema IRO. Quando há uma proposta pendente, aparece um card amarelo com botão "Ver" para aprovar ou rejeitar. Propostas aprovadas modificam o SOUL.md e MEMORY.md automaticamente — com backup garantido.
SAPP · Segurança
Três contadores: N1 (Autônomo) — ações executadas sem confirmação. N2 (Confirmado) — ações que aguardaram aprovação do owner. N3 (Bloqueado) — ações recusadas por violarem o perímetro ou serem irreversíveis. Ver log completo em Segurança → Log SAPP.
Botões de Ação
💬 Chat — conversa diretamente com o agente pelo dashboard (usa o modelo configurado no banco via OpenRouter).
✓ Tarefas — vai para a lista de tarefas filtrada por este agente.
📋 Logs — vai para o log filtrado por este agente.
📋 The Call — abre o formulário THE CALL pré-preenchido com os dados do agente para revisão ou atualização completa.
💓 — abre o HEARTBEAT.md do agente.
➕ Como Adicionar um Novo Agente
1
Clique em "+ Novo Agente" na página de Agentes. O formulário THE CALL abre com 8 etapas.
2
Preencha os 8 blocos: Identidade → Backstory → Personalidade → Competências → Missão → Pessoa Atendida → Canais → Configuração Técnica.
3
Na etapa 8 defina: ID (slug único), cor, modelo LLM, token Telegram e email. O ID não pode ser alterado depois.
4
Clique em "Criar Agente". O sistema cria os 12 arquivos markdown, provisiona o workspace, registra o webhook do Telegram e reinicia o OpenClaw.
5
O agente aparece no dashboard com status Onboarding. Envie uma mensagem pelo Telegram para iniciar o processo de humanização.
✏️ Como Editar um Agente
Editar arquivos MD: Clique em qualquer pill de arquivo no card. Um editor de texto abre inline. Salve — o arquivo é atualizado imediatamente no workspace.

Arquivos sem restart: MEMORY.md, MEMORY_MID.md, MEMORY_LONG.md, HUMANIZED.md, TASKS.md, HEARTBEAT.md, EVOLUTION.md entram em vigor na próxima sessão do agente.

Arquivos com restart: SOUL.md, IDENTITY.md, AGENTS.md, TOOLS.md, ONBOARDING.md — após salvar, o OpenClaw reinicia automaticamente para carregar as mudanças.

The Call (revisão completa): Clique no botão "📋 The Call" no card do agente para abrir o formulário pré-preenchido com os dados atuais. Use para realinhar o agente com um novo contexto ou papel.
🗑️ Como Excluir um Agente
A exclusão de agente é uma operação de Nível 3 SAPP — requer confirmação dupla e não pode ser desfeita automaticamente.

Via dashboard (em breve): botão de exclusão com confirmação em dois passos.

Via banco de dados: UPDATE agents SET active = false WHERE id = 'slug'

O workspace em /home/openclaw/.openclaw/workspace-{slug} é preservado para auditoria. Remover manualmente apenas após confirmar que todos os dados foram preservados em backup.
🔍 Filtros de Agentes
Na página de Agentes, use os botões de filtro para segmentar a visualização:

Todos — exibe todos os agentes cadastrados.
● Online — apenas agentes com active = true no banco.
○ Offline — agentes inativos.
Por linha ARVA (Professional, Autism, SDR, Support, Custom) — filtra pelo campo arva_line no banco ou pelo role do agente.
🧠 SMT · IRO · SAPP — Resumo
🧠 SMT
Smart Memory Transhipment. Gerencia 6 camadas de memória com promoção automática. Resolve apagões entre canais (Session Bridge) e abandono de tarefas longas (Execution State).
📈 IRO
Improving Relationship with Owner. Observa padrões de uso e propõe melhorias de comportamento semanalmente. Mudanças só aplicadas com aprovação explícita do owner.
🔒 SAPP
Security Agent Process Protocol. Classifica toda ação em N1 (autônomo), N2 (confirmar) ou N3 (bloqueado). Inclui Anti-Injection para conteúdo externo.
